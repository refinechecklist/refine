REFINE Checklist

Section Key Reporting Items Response Page / Section
1. Model Specification
1.1 Model name, vendor, version, release and training cutoff date (e.g., GPT-4, Gemini 2.5)
1.2 Architecture (e.g., transformer, diffusion), pretraining, fine-tuning, or instruction tuning (e.g., MedPaLM, BioGPT)
1.3 Modality support (e.g., text-only, text+image)
1.4 Language capabilities (e.g., English-only, multilingual)
1.5 Model access (e.g., ChatGPT web, Gemini API, local installation) and any limitations
2. Prompt Design
2.1 Purpose/context of use (e.g., radiology report simplification, clinical question answering)
2.2 Prompt format (e.g., templates with placeholders, open-ended questions), constraints (e.g., word limits, style guides), and length
2.3 Multimodal input details (e.g., PNG chest X-ray, PDF discharge summary)
2.4 Interaction style (e.g., one-turn query, 3-turn chatbot conversation), memory usage
2.5 Prompting strategy (e.g., zero-shot with task instruction, few-shot with examples)
2.6 Prompt engineering history (e.g., version 1: template A; version 2: modified for clarity) and tuning/test data separation
3. Stochasticity Control
3.1 Generation parameters: temperature, top-k, top-p, max token, penalties, seed
3.2 Image generation: inference steps, scheduler, resolution, guidance scale
3.3 User roles and prompt attempts (e.g., 2 users, 3 prompts per case)
3.4 Output selection (e.g., expert-reviewed, randomly selected, consensus-based)
3.5 Reproducibility constraints and uncertainty reporting
4. Dataset Integrity
4.1 Dataset name, source, and access type
4.2 Ethics and consent statements
4.3 Prior usage and literature citation
4.4 Dataset composition and source heterogeneity
4.5 Preprocessing and alignment
4.6 Missing data handling
4.7 Sample characteristics (e.g., age, sex, disease, imaging modality)
4.8 Evaluation reference standard and annotator qualifications
5. Output Evaluation
5.1 Evaluation methods (e.g., Likert scale ratings, automated scoring)
5.2 Human rater details (e.g., number, specialty, experience level)
5.3 Subgroup-level analysis (e.g., age groups, disease categories)
5.4 Performance metrics (e.g., BLEU, F1, AUROC, readability index, hallucination detection)
5.5 Prompt revisions and performance effects (e.g., v1 vs. v2 differences)
5.6 Version comparisons (e.g., GPT-4 vs. GPT-4o) and temporal performance variation
