REFINE Checklist

Section Key Reporting Items Response Page / Section
1. Model Specification
1.1 Model name, vendor, version, release, and training cutoff date (e.g., GPT-4, Gemini 2.5)
1.2 Architecture (e.g., transformer, diffusion), pretraining, fine-tuning, or instruction tuning details
1.3 Modality support details (e.g., text-only, text+image), with fusion strategies (e.g., early, intermediate, late)
1.4 Language capabilities (e.g., English-only, multilingual)
1.5 Model access (e.g., ChatGPT web, Gemini API, local installation)
1.6 Intended clinical application/scope of use (e.g., diagnostic support, clinical question answering, workflow improvement)
1.7 Model limitations, explicit clinical non-use cases, and consideration of potential misuse (with attention to patient harm or health system risk)
1.8 Computational resources for training/fine-tuning and deployment (if applicable)
2. Prompt Design
2.1 Prompt format (e.g., templates with placeholders, open-ended questions), constraints (e.g., word limits, style guides), and length
2.2 Input details (e.g., chest X-ray, discharge summary) along with characteristics (e.g., file format (PNG, PDF), resolution)
2.3 Interaction style (e.g., one-turn query, 3-turn chatbot conversation) and memory usage
2.4 Prompting strategy (e.g., zero-shot with task instruction, few-shot with examples)
2.5 Prompt engineering history (e.g., version 1: template A; version 2: modified for clarity)
2.6 Integration of relevant patient clinical context (e.g., age, sex, key comorbidities, relevant medical history) into prompts, how this contextual information was selected, standardized, and its source
3. Stochasticity Control
3.1 Generation parameters: e.g., temperature (e.g., 0.7), top-k (e.g., 40), top-p (e.g., 0.9), max token (e.g., 2048), penalties, seed
3.2 Image generation (if applicable): e.g., inference steps (e.g., 50), scheduler (e.g., DDIM), resolution (e.g., 512Ã—512), guidance scale
3.3 User roles and prompt attempts (e.g., 2 users, 3 prompts per case)
3.4 Output selection (e.g., expert-reviewed, randomly selected, consensus-based)
3.5 Reproducibility constraints and uncertainty reporting (e.g., seed limitation in API, variability across outputs)
4. Dataset Integrity
4.1 Dataset name, specific clinical sources (e.g., department electronic health records, clinical trial data), access type (e.g., public, restrictive, private), and license details for publicly available datasets, if applicable
4.2 Ethics and consent statements (e.g., IRB approval, waiver)
4.3 Prior data usage and literature citation
4.4 Dataset composition and source heterogeneity (e.g., single-site vs. multi-center)
4.5 Preprocessing and alignment details at dataset level (e.g., text de-identification/anonymisation, image pairing)
4.6 Missing data handling (e.g., imputation, exclusion)
4.7 Sample characteristics (e.g., age, sex, disease, imaging modality) and analysis of potential input-level data representational biases
4.8 Evaluation reference standard and annotator qualifications (e.g., radiologist consensus)
4.9 Clear separation of tuning/test data
5. Output Evaluation
5.1 Evaluation methods (e.g., Likert scale ratings, automated scoring)
5.2 Human rater (e.g., number, specialty, experience level) and reliability analysis details
5.3 Subgroup-level analysis (e.g., age groups, disease categories)
5.4 Performance metrics (e.g., BLEU, ROUGE, F1, AUROC, readability index, hallucination)
5.5 Performance effects of prompt revisions (e.g., v1 vs. v2 differences)
5.6 Version comparisons (e.g., GPT-4 vs. GPT-4o) and temporal performance variation
5.7 Assessment of model output bias and fairness across clinically relevant groups (e.g., fairness metrics, qualitative review)
5.8 Details on safety testing protocols for harmful, medically inaccuracy, or inappropriate output
5.9 Methods for explainability/interpretability of model outputs (if used) and their limitations
5.10 Comparison of clinically relevant non-FM baselines or existing clinical standards (if applicable; e.g., expert human performance, established guidelines)
5.11 Assessment of potential clinical utility or added value (e.g., diagnostic accuracy improvement, time-saving for clinicians, improved patient understanding)
